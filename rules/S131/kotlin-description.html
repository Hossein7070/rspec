The requirement for a final <code>default</code> clause is defensive programming. The clause should either take appropriate action, or contain a suitable comment as to why no action is taken.

Noncompliant Code Example

when(param) { // missing else
  1 -> doSomething()
  2 -> doSomethingElse()
}

Compliant Solution

when(param) {
  1 -> doSomething()
  2 -> doSomethingElse()
  else -> error("myMessage")
}

Exceptions

If the <code>when</code> clause is used as an expression, Kotlin compiler already enforces that all possible cases are covered with branch conditions. So no issue will be raised in this case.

Example:

enum class Day {
    SUNDAY, MONDAY
}
...
val myVal = when(day) {
  MyEnum.SUNDAY -> 1
  MyEnum.MONDAY -> 2
}

See

  • MITRE, CWE-478 - Missing Default Case in Switch Statement
  • CERT, MSC01-C. - Strive for logical completeness
