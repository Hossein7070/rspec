Using <code>TOP</code> in a <code>SELECT</code> without ordering the results from which the "top" results are chosen will return a seemingly random set of rows, and is surely a mistake. The same random behavior also occurs when using <code>TOP</code> in a <code>DELETE</code>, <code>INSERT</code>, <code>UPDATE</code> and <code>MERGE</code>.

Noncompliant Code Example

SELECT TOP 10 -- Noncompliant selects 10 random rows
  fname, lname, city
  FROM people
  WHERE city IS NOT NULL;

DELETE TOP (10) -- Noncompliant deletes 10 random rows
  FROM PurchaseOrder
  WHERE DueDate < '20020701';

Compliant Solution

SELECT TOP 10
  fname, lname, city
  FROM people
  WHERE city IS NOT NULL
  ORDER BY birthdate;

DELETE
  FROM PurchaseOrder
  WHERE OrderID IN (
    SELECT TOP 10
      OrderID
      FROM PurchaseOrder
      WHERE DueDate < '20020701'
      ORDER BY DueDate ASC
  );
