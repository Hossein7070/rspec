When boxed type <code>java.lang.Boolean</code> is used as an expression it will throw <code>NullPointerException</code> if the value is <code>null</code> as defined in Java Language Specification ยง5.1.8 Unboxing Conversion.

It is safer to avoid such conversion altogether and handle the <code>null</code> value explicitly.

Noncompliant Code Example

Boolean b = getBoolean();
if (b) {  // Noncompliant, it will throw NPE when b == null
  foo();
} else {
  bar();
}

Compliant Solution

Boolean b = getBoolean();
if (Boolean.TRUE.equals(b)) {
  foo();
} else {
  bar();  // will be invoked for both b == false and b == null
}
