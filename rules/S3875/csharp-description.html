The use of <code>==</code> to compare to objects is expected to do a reference comparison. That is, it is expected to return <code>true</code> if and only if they are the same object instance. Overloading the operator to do anything else will inevitably lead to the introduction of bugs by callers. Exceptions

  • Classes with overloaded <code>operator +</code> or <code>operator -</code> methods are ignored.
  • Classes that implement <code>IComparable<T></code> or <code>IEquatable<T></code> most probably behave as a value-type objects and so are ignored.
