<code>PERFORM</code> is used to execute a paragraph located somewhere in the program and then, once executed, the execution flow will continue on the line following the PERFORM statement. This is the expected behaviour that can be broken if a <code>GO TO</code> is added in the called paragraph. When mixing <code>PERFORM</code> and <code>GO TO</code> you can quickly be lost on the execution flow and finally donâ€™t get the one you expect. For this reason, calling <code>PERFORM</code> with paragraphs that used <code>GO TO</code> should be avoided.

Noncompliant Code Example

PROCEDURE DIVISION.
DISPLAY-9-LETTERS.
    PERFORM ABC.
    DISPLAY "END OF DISPLAY-9-LETTERS".
    STOP RUN.

    ABC.
        DISPLAY "ABC".
        GO TO XYZ.

    DEF.
        DISPLAY "DEF".

    XYZ.
        DISPLAY "XYZ".

Compliant Solution

PROCEDURE DIVISION.
DISPLAY-9-LETTERS.
    PERFORM ABC.
    PERFORM DEF.
    PERFORM XYZ.
    DISPLAY "END OF DISPLAY-9-LETTERS".
    STOP RUN.

    ABC.
        DISPLAY "ABC".

    DEF.
        DISPLAY "DEF".

    XYZ.
        DISPLAY "XYZ".
