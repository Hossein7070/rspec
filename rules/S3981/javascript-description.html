The size of a collection and the length of an array are always greater than or equal to zero. So testing that a size or length is greater than or equal to zero doesnâ€™t make sense, since the result is always <code>true</code>. Similarly testing that it is less than zero will always return <code>false</code>. Perhaps the intent was to check the non-emptiness of the collection or array instead.

Noncompliant Code Example

if (someMap.size < 0) {...} // Noncompliant
