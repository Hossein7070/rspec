Classes defined inside functions, called local classes, are only visible within those functions. They have advantages in some situations, but if theyâ€™re just being used as functors, lambdas are preferred.

This rule raises an issue when a <code>class</code> or <code>struct</code> is defined inside a function.

Noncompliant Code Example

void doSomething() {

  struct something {  // Noncompliant
    int a;
  };

  //...
}

Compliant Solution

struct something {  // Noncompliant
  int a;
};

void doSomething() {

  //...
}
