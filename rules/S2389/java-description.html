When a cycle exists between classes during their <code>static</code> initialization, the results can be unpredictable because they depend on which class was initialized first.

Noncompliant Code Example

public class A {
  public static int a = B.b + 1;  // Noncompliant; sometimes a = 1, others a = 2
}

public class B {
  public static int b = A.a + 1;  // Noncompliant; sometimes b = 1, others b = 2
}

  • CERT, DCL00-J. - Prevent class initialization cycles
