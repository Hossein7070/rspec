If none of the font names defined in a <code>font</code> or <code>font-family</code> declaration are available on the browser of the user, the browser will display the text using its default font. Itâ€™s recommended to always define a generic font family for each declaration of <code>font</code> or <code>font-family</code> to get a less degraded situation than relying on the default browser font. All browsers should implement a list of generic font matching these families: <code>Serif</code>, <code>Sans-serif</code>, <code>cursive</code>, <code>fantasy</code>, <code>Monospace</code>.

Noncompliant Code Example

a {
  font-family: Helvetica, Arial, Verdana, Tahoma; /* Noncompliant; there is no generic font family in the list */
}

Compliant Solution

a {
  font-family: Helvetica, Arial, Verdana, Tahoma, sans-serif;
}
