Having all branches in a <code>when</code> or <code>if</code> chain with the same implementation is an error. Either a copy-paste error was made and something different should be executed, or there shouldnâ€™t be a <code>when</code>/<code>if</code> chain at all.

Noncompliant Code Example

if (b == 0) {  // Noncompliant
     doOneMoreThing()
} else {
     doOneMoreThing()
}

when (i) {  // Noncompliant
    1 -> doSomething()
    2 -> doSomething()
    3 -> doSomething()
    else -> doSomething()
}

Exceptions

This rule does not apply to <code>if</code> chains without <code>else</code>-s, or to <code>when</code>-es without <code>else</code> clauses.

if (b == 0) {
    doOneMoreThing()
} else if (b == 1) {
    doOneMoreThing()
}
